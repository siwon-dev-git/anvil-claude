# Axioms

The three axioms are non-negotiable foundations of Anvil governance.

## 1. Environment is intelligence

Entity intelligence < Substrate quality. Investment priority: Environment > Protocol > Entity.

The quality of the project's governance structure (constitution, ADR, FMEA, gate chain) determines how effectively any agent can work within it. Improving the environment yields greater returns than improving the agent.

## 2. Design physics, not control

Create environments (gravity fields) where commands are unnecessary. Good structure makes bad outcomes difficult and good outcomes natural.

Gate scripts embody this: they don't "tell" the agent to check quality — they enforce it via exit codes.

## 3. Success is becoming unnecessary

Trust Level rises -> Creator intervention decreases. The system should become increasingly autonomous while maintaining safety through constitution and gate chain.

Self-sufficiency stages: A(dependent) -> B(assisted) -> C(balanced) -> D(growing)
